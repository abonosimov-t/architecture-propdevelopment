# Задание 3. Внешние интеграции
## 1. Требования к безопасности
### Защита данных:
* Все персональные данные должны храниться и обрабатываться в соответствии с ФЗ-152
* Биометрические данные (фотографии лиц) должны храниться в зашифрованном виде
* Данные о номерах автомобилей должны быть защищены как персональные данные
* Все коммуникации между системами должны быть зашифрованы с использованием TLS 1.3
* Необходимо внедрить систему логирования всех действий с биометрическими данными
* Разграничение доступа:
    * Каждый житель должен иметь доступ только к своим устройствам
    * Необходимо реализовать механизм делегирования прав доступа (например, для гостей)
    * Администраторы системы должны иметь строго определенные роли
    * Партнёр не должен иметь прямого доступа к базе данных жильцов
## 2. Протоколы аутентификации и авторизации
* Аутентификация:
    * Использование OAuth 2.0 для аутентификации между сервисами
    * Двухфакторная аутентификация для доступа к управлению устройствами
    * JWT-токены для сессий пользователей
    * Отдельные API-ключи для каждого сервиса партнёра
* Авторизация:
    * Использование RBAC (Role-Based Access Control)
    * Внедрение системы управления политиками доступа
    * Механизм отзыва прав доступа в реальном времени
    * Ограничение времени действия токенов доступа
## 3. Организация взаимодействия между системами
* Архитектурные решения:
    * Создание отдельного API-шлюза для взаимодействия с партнёрскими сервисами
    * Реализация паттерна Circuit Breaker для обработки сбоев
    * Внедрение очередей сообщений для асинхронного взаимодействия
    * Использование микросервисной архитектуры для новых компонентов
* Интеграционные требования:
    * Все API должны быть версионированы
     Необходимо реализовать механизм rate limiting
    * Обязательное ведение журнала всех запросов
    * Мониторинг доступности и производительности API
* Безопасность API:
    * Использование HTTPS для всех endpoint'ов
    * Валидация всех входящих данных
    * Реализация механизма защиты от DDoS-атак
    * Регулярное сканирование уязвимостей API
