# Задание 2. Проверочный лист


## 1. Анализ систем с точки зрения безопасности:

Основные компоненты, требующие особого внимания:
- Витрина продаж с персональными данными клиентов
- Client-mart-app (онлайн-сделки) - работа с конфиденциальными данными
- CRM системы (клиентские данные)
- Tenant-core-app (сервисы для собственников)
- Финансовые системы
- API интеграции с партнерами

Ключевые риски:
- Множественные точки регистрации клиентов без координации
- Неконтролируемые потоки данных между системами
- Проблемы с API партнеров (утечка персональных данных)
- Отображение данных других клиентов
- Отсутствие единой политики безопасности

## 2. Выбор разделов для проверочного листа:

Исходя из проблем компании, стоит сфокусироваться на следующих разделах:

### 1. Безопасность данных
* В компании нарушен контроль данных с несколькими несогласованными точками регистрации клиентов
* Есть проблемы с отображением данных других клиентов в личных кабинетах
* Компания планирует развивать направление обработки данных (BI, ML, AI)
* Недавний инцидент с утечкой данных у конкурента вызывает беспокойство руководства
* Необходимость соответствия законодательству в области защиты персональных данных

### 2. Управление доступом
* Отсутствие системного контроля за внутренними потоками данных
* Проблемы с разграничением доступа в API партнеров
* Возможность доступа к данным клиентов других управляющих компаний
* Отсутствие ограничений на доступ к данным со стороны систем обработки

### 3. Управление инцидентами
* Нарушен процесс передачи информации между командами после смены руководства
* Единственный специалист по ИБ не получает информацию обо всех решениях и изменениях
* Необходимость предотвращения инцидентов, подобных утечке у конкурента
* Отсутствие системного подхода к обнаружению и реагированию на инциденты

### Разделы, которые можно исключить или отложить на второй этап:
### 1. Защита инфраструктуры 
* Базовая защита уже реализована (межсетевой экран, гибридная инфраструктура)
* Нет явных проблем с инфраструктурной безопасностью в описании
* Текущие проблемы больше связаны с управлением данными и доступом
### 2. Специфические аспекты 
* Более общие вопросы, не связанные напрямую с текущими проблемами компании
* Физическая безопасность не является приоритетной в контексте описанных проблем
* Управление мобильными устройствами не упоминается как критическая проблема
